数据库优化 添加索引（好好学习吧） 索引，联合索引    尽可能在内存中读取数据，不要在磁盘中读取，速度慢
那么查找任何哪怕只是一条特定的数据都会进行一次全表扫描，如果一张表的数据量很大而符合条件的结果又很少，那么不加索引会引起致命的性能下降。
1.创建索引/组合索引：尽量在where 或者order by后面的字段加索引，提升查询效率
2.select查询的时候不要使用* ，用到的字段才添加
3.用varchar/nvarchar 代替 char/nchar
4.select 能用between就不用in

复合索引
比如有一条语句是这样的：select * from users where area=’beijing’ and age=22;
如果我们是在area和age上分别创建单个索引的话，由于mysql查询每次只能使用一个索引，所以虽然这样已经相对不做索引时全表扫描提高了很多效率，但是如果在area、age两列上创建复合索引的话将带来更高的效率。如果我们创建了(area, age,
salary)的复合索引，那么其实相当于创建了(area,age,salary)、(area,age)、(area)三个索引，这被称为最佳左前缀
特性。因此我们在创建复合索引时应该将最常用作限制条件的列放在最左边，依次递减。

sql:EXPLAIN select * from lzy_user_address where address LIKE '测%' ORDER BY consignee asc,address asc;


/**ALTER TABLE table_name ADD INDEX index_name (column_list)**/
/**普通索引**/
ALTER TABLE lzy_user_address ADD INDEX index_address (address);

/**查看索引**/
show INDEX from lzy_user_address;


/**组合索引 相当于创建了(consignee,address)、(consignee)两个索引**/
alter TABLE lzy_user_address ADD INDEX index_all(consignee,address);
